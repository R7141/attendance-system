# 签到系统 (Sign In System)

这是一个包含 **Web 管理端（教师端）**、**微信小程序端（学生端）** 和 **Go 后端** 的综合签到系统，支持多种签到防作弊校验及 AI 离群值分析。

## 系统架构与角色

1. **Web 管理端**：教师或管理员使用，用于管理组织、课程、班级名单、编辑座位图、发起签到、实时监控签到进度、导出考勤报表以及查看 AI 离群预警。
2. **微信小程序端**：学生使用，绑定身份后，通过扫描教师端生成的动态二维码进行签到。
3. **Go 后端**：基于 Gin 框架和 GORM 编写，提供全套的 RESTful API 支持，负责身份认证、数据持久化以及核心的签到规则校验和数据分析算法。

***

## 核心业务流程

### 1. 登录与认证流程 (Authentication)

- **教师端 (Web)**：通过 `/api/login` 账号密码登录，获取 JWT Token，并在后续请求头中携带 `Authorization: Bearer <token>` 进行身份验证。
- **学生端 (小程序)**：
  1. 调用 `wx.login` 获取临时 `code`。
  2. 请求后端 `/wx/login` 接口，后端向微信服务器换取该用户的唯一标识 `OpenID` 并签发专属的 WxToken。
  3. 学号与姓名由小程序端本地存储（不落库到服务器），允许用户在本地修改/重新填写。
  4. 后续签到时请求头携带 WxToken，后端解析出 `OpenID` 用于同场次防重复签到；学号/姓名来自请求体。

### 2. 课程与考勤准备 (Preparation)

1. **创建场地与座位表**：教师在系统中创建场地（Room），并通过内置的 Canvas 座位编辑器绘制教室座位分布，保存时前端会生成 JSON 格式的 `seat_pos` 坐标数据，后端落库。
2. **创建课程**：教师创建课程，可配置关联的“班级名单”或“独立名单”。
3. **高级防作弊配置**：在课程设置中，教师可按需开启以下校验项：
   - **GPS 定位校验**：设置中心经纬度与允许半径。
   - **固定座位**：将学生与座位一对一绑定，签到时仅允许在分配好的座位上签到。
   - **Wi-Fi (BSSID) 校验**：限制只能连接特定路由器的学生签到。

### 3. 签到场次发起流程 (Session Flow)

1. 教师在 Web 端进入课程，点击“发起签到”。
2. 系统调用 `/api/sign_sessions` (POST) 创建新的活跃场次（`SignSession`），生成场次 ID。
3. Web 端通过 `/api/qrcode` 实时生成动态签到二维码（包含 `session_id`、时间戳和座位信息），并在大屏幕上展示。二维码默认带有过期时间（TTL）以防截图外传。

### 4. 学生签到核心流程 (Sign-In Logic)

1. 学生使用微信小程序扫描屏幕上的二维码，解析出其中的参数。
2. 小程序收集本地设备信息（如 GPS 坐标、BSSID）后，向后端 `/signin` 接口发起 POST 请求。
3. **后端校验防作弊链路 (`PostSignIn`)**：
   - **身份解析**：从 Token 提取 `OpenID`，用于同一微信号在同一场次的防重复签到校验。
   - **重复拦截**：检查该场次中该 `StudentID` 或 `OpenID` 是否已签到。
   - **座位锁定**：检查请求座位是否被他人占用；若课程开启了“固定座位”，则严格校验所选座位是否与排座表一致。
   - **GPS 校验**：若课程开启了 GPS 签到，计算上报经纬度与目标经纬度的球面距离，超出半径则拦截；若小程序端定位失败，则尝试调用第三方 IP 库 (`/geo/ip`) 进行基站定位兜底。
   - **BSSID 校验**：若配置了路由器白名单，检查上报的 BSSID。
4. 校验全部通过后，向 `sign_ins` 表写入记录，状态为 `success`。
5. **触发异步/同步钩子**：调用 `generateSignInAnomalies` 触发 AI 异常检测。

### 5. AI 离群值分析流程 (AI Outlier Detection)

每次签到成功后，后端会拉取该学生在当前课程**最近 30 次**的历史签到数据，利用算法识别异常行为并落库至 `sign_anomaly_alerts` 表。

- **时间离群检测 (1D K-Means)**：
  - 提取历史“签到时间相对于开场时间的分钟差”。
  - 利用一维 K-Means 算法 (`k=2`) 对历史时间进行聚类，找到常规的主时间簇。
  - 如果本次签到时间严重偏离该主簇（超过 3 倍标准差或绝对阈值），则生成“时间异常”预警（如：平时都在开场前签到，今天突然迟到 20 分钟）。
- **座位离群检测 (DBSCAN 空间聚类)**：
  - 解析房间的 `seat_pos` 坐标，提取该学生历史所坐座位的真实 `(x, y)` 坐标。
  - 利用 DBSCAN 密度聚类算法（自适应计算 `eps` 参数），识别出该学生的“常用座位区域/簇”。
  - 如果本次选择的座位坐标偏离了其历史的主簇区域，则生成“座位选择异常”预警。若坐标数据缺失，则自动回退到基于频次的“常用座位占比 ≥70%”判定逻辑。

### 6. 数据统计与导出 (Export & Analytics)

- **实时监控**：教师端不断轮询 `/api/sign_sessions/:id/signs` 获取最新数据，在 Canvas 座位图上实时渲染签到学生的姓名，并可查看请假标记。
- **热力图分析**：前端请求 `/api/ai/seat_heatmap`，后端聚合多场次数据计算各座位的上座率，前端以颜色渐变渲染到座位图上。
- **导出 Excel**：教师端点击单场次的导出，后端调用 `excelize` 库，根据班级名单交叉比对出“已签到”与“未签到”明细，生成 `.xlsx` 文件提供下载 (`/api/attendance/export_session`)。
